# Hoja de Ejercicios 2: Fundamentos de HTML II 🏗️

Este directorio contiene las soluciones a la segunda hoja del bootcamp. El objetivo principal ha sido dominar la estructuración de datos complejos y la creación de interfaces interactivas utilizando exclusivamente HTML5 nativo, prestando especial atención a la semántica y la accesibilidad.

## 📝 Detalle de los Ejercicios

El trabajo se ha dividido en distintas fases progresivas:

### Formularios y Validación Nativa
* **Ejercicio 1 - Ficha básica:** Creación de un formulario de inscripción aplicando la relación fundamental entre `<label>` (con su atributo `for`) e `<input>` (con su `id` y `name`).
* **Ejercicio 2 - Validación:** Implementación de restricciones nativas en el navegador usando atributos como `required`, `minlength`, `pattern` y tipos de input específicos (`email`, `password`).
* **Ejercicio 3 - Agrupación Semántica:** Uso de `<fieldset>` y `<legend>` para estructurar bloques de opciones de selección única (`radio`) y múltiple (`checkbox`).
* **Ejercicio 4 - Búsqueda y Filtros:** Diseño de una interfaz de filtrado simulada.

### Tablas de Datos
* **Ejercicio 5 - Tabla Básica:** Construcción de información tabular accesible usando `<table>`, `<caption>`, `<thead>`, `<tbody>`, y definiendo los encabezados de columna con `<th scope="col">`.
* **Ejercicio 6 - Tabla Avanzada:** Profundización en las relaciones de datos definiendo la primera celda de cada fila como un encabezado de fila mediante `<th scope="row">`.

### Elementos Nativos Interactivos y de Estado
* **Ejercicio 7 - Preguntas Frecuentes (FAQ):** Creación de un sistema de acordeones desplegables sin usar JavaScript, empleando las etiquetas `<details>` y `<summary>`.
* **Ejercicio 8 - Panel de Progreso:** Diferenciación conceptual y práctica entre tareas que avanzan hacia una meta (`<progress>`) e indicadores de estado actuales (`<meter>`).
* **Ejercicio 9 - Diálogos e Incrustaciones:** Integración de ventanas modales nativas con `<dialog>` y contenido externo con `<iframe>` (asegurando la accesibilidad con el atributo `title`).

### Auditoría e Integración
* **Ejercicio 10 - Refactorización:** Auditoría de un formulario defectuoso para corregir errores críticos de accesibilidad, semántica de tipos y estructura de envío de datos.
* **Ejercicio 11 - Proyecto Integrador final:** Construcción de una interfaz completa ("Inscripción y seguimiento del bootcamp") que combina todos los elementos de la hoja: un formulario robusto, una tabla bidireccional, un panel de estado nativo y un bloque de preguntas frecuentes.

## 🛠️ Tecnologías y Flujo
* HTML5 Semántico y Accesible.
* Git & GitHub (flujo de trabajo por ramas `feature/` y Conventional Commits).
